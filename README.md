# Leveraged Rotation Strategy — Backtest Dashboard & Live Alerts

An interactive dashboard that **simulates, backtests, and tunes** a TQQQ/SQQQ leveraged
"engine" + a **swappable income/anchor sleeve** (JEPQ/JEPI/QYLD/SCHD/BIL/ADX), with an
optional **whipsaw filter** (ADX / Efficiency-Ratio → TLT), **after-tax** modeling, and
**email + Telegram alerts** when a rebalance is due. Benchmarked vs **QQQ** and **TQQQ**.

## Repository layout

```
src/engine.js        Backtest engine (pure, dependency-free)
src/ui.js            Dashboard UI logic
src/template.html    HTML + CSS shell (placeholders for engine/ui/data)
data/strategy_data.json   Dividend-adjusted daily prices 2010–2026 + TQQQ OHLC (Yahoo)
build.js             Assembles src + data -> index.html (and public/index.html)
test/test_engine.js  Engine unit tests (buy&hold parity, metrics, tax, rebalance modes)
test/test_ui.js      Headless DOM smoke tests (jsdom)
ci/notify_failure.py CI failure -> emails the repo admin
alerts/check_signal.py     Live signal checker + email/Telegram sender (stdlib only)
alerts/strategy_config.json  Strategy parameters for the alert workflow
.github/workflows/ci.yml      Build -> test -> deploy (only if tests pass) -> email on fail
.github/workflows/alerts.yml  Scheduled rebalance alerts
index.html           Built dashboard (generated by build.js; open it directly to use)
```

## Develop locally

```
npm install          # installs jsdom (for UI tests)
npm run build        # regenerate index.html from src/ + data/
npm test             # build + run engine and UI test suites
```

Edit the dashboard in `src/`. Run `npm run build` to refresh `index.html`. Never hand-edit
`index.html` — it is generated.

## Continuous integration & deployment (CI/CD)

`.github/workflows/ci.yml` runs on every push and pull request to `main`:

1. **Build** `index.html` from source.
2. **Test** — engine unit tests, headless UI smoke tests, and an alert-script compile check.
3. **Deploy** to GitHub Pages **only if every test passes**. If any test fails, the deploy
   steps are skipped (the change is **not** published) and `ci/notify_failure.py` **emails the
   repo admin** the commit, run link, and failing test output.

So a change that breaks the dashboard never reaches the live site. Pull requests run the same
tests (turn on branch protection requiring the "CI" check to block merges of failing PRs).

Enable it: repo → **Settings → Pages → Source: GitHub Actions** (one-time), then push.

## Live rebalance alerts (email + Telegram)

`alerts/check_signal.py` reuses the exact signal logic, pulls fresh prices from Yahoo, and
emails/Telegrams you **only when the target allocation changes**.

### Repository secrets — Settings → Secrets and variables → Actions

| Secret | Used by | What it is |
|---|---|---|
| `TELEGRAM_BOT_TOKEN` | alerts | Token from **@BotFather** |
| `TELEGRAM_CHAT_ID` | alerts | Your chat id |
| `SMTP_USER` | alerts + CI | Sending Gmail address |
| `SMTP_PASS` | alerts + CI | Gmail **App Password** (16 chars) |
| `ALERT_TO` | alerts (+ CI fallback) | Alert destination address |
| `CI_ADMIN_EMAIL` | CI | Where CI failure emails go (falls back to `ALERT_TO`) |
| `SMTP_HOST` / `SMTP_PORT` | *(optional)* | default `smtp.gmail.com` / `587` |

- **@BotFather** → `/newbot` → copy token; message the bot, then open
  `https://api.telegram.org/bot<TOKEN>/getUpdates` for your `chat.id`.
- Gmail **App Password**: enable 2-Step Verification → *Security → App passwords*.

When it runs: Friday 15:45 ET by default (`alerts/check_signal.py --force` to test;
*Actions → Strategy rebalance alerts → Run workflow*).

## Dashboard highlights

Swappable 30% anchor; weekly/monthly/daily/**signal-driven** rebalancing; next-open vs
~15-min-before-close execution; **taxable vs tax-advantaged** with holding-period short/long-term
capital-gains tax (default taxable — the after-tax hit on weekly leverage is large); whipsaw
filter; full risk/reward metrics; equity/drawdown/allocation/annual charts; saved-version
view/edit/save; and checkboxes to include/exclude versions and benchmarks in the comparison.

## Data & proxies

Dividend-adjusted daily closes from Yahoo Finance. Anchor funds launched at different times
(JEPQ 2022, JEPI 2020, QYLD 2013, SCHD 2011; BIL & ADX predate 2010); before a fund's launch
its return is a **calibrated proxy** (R² ≈ 0.75–0.93), marked on the charts. Taxes use
average-cost basis, yearly payment, no wash-sale rule — an approximation, **not tax advice**.

## Disclaimer

Educational backtesting tool. Past and simulated performance does not predict future results.
